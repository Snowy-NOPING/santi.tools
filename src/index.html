what does remix do?

remix fixes any issues with the file container, such as missing time info. it helps increase compatibility with old software, such as vegas pro and windows media player.

how does it work?

remuxing takes existing coded data and copies it over to a new media container. it's lossless: media data doesn't get re-encoded.

on-device processing

cobalt remuxes files locally. files never leave your device, so processing is nearly instant.

settings

video quality ⊘

if preferred video quality isn't available, next best is picked instead.

preferred youtube video codec ⊘

h264: best compatibility, average quality. max quality is 1080p.
av1: best quality and efficiency. supports 8k & HDR.
vp9: same quality as av1, but file is ~2x bigger. supports 4k & HDR.

high efficiency video codec ⊘

allow h265 for videos

allows downloading videos from platforms like tiktok in higher quality at cost of compatibility.

twitter/x ⊘

convert looping videos to GIF

GIF conversion is inefficient, converted file may be obnoxiously big and low quality.

audio format ⊘

all formats but "best" are converted from the source format, there will be some quality loss. when "best" format is selected, the audio is kept in its original format whenever possible.

audio bitrate ⊘

bitrate is applied only when converting audio to a lossy format. cobalt can't improve the source audio quality, so choosing a bitrate over 128kbps may inflate the file size with no audible difference.

youtube audio track ⊘

prefer better quality

cobalt will try to pick highest quality audio in audio mode. it may not be available depending on youtube's response, current traffic, and server status.

tiktok ⊘

download original sound

cobalt will download the sound from the video without any changes by the post's author.

filename style ⊘

Video Title - Video Author (1080p, h264).mp4 video file preview
Audio Title - Audio Author.opus audio file preview

filename style will only be used for files tunneled by cobalt. some services don't support filename styles other than classic.

file metadata ⊘

disable file metadata

title, artist, and other info will not be added to the file.

community instances ⊘

use a custom processing server

cobalt will use a custom processing instance if you choose to. even though cobalt has some security measures in place, we are not responsible for any damages done via a community instance, as we have no control over them.

please be mindful of what instances you use and make sure they're hosted by people you trust.

custom instance url ⊘

instance access key ⊘

use an instance access key

cobalt will use this key to make requests to the processing instance instead of other authentication methods. make sure the instance supports api keys!

tunneling ⊘

always tunnel files

cobalt will hide your ip address, browser info, and bypass local network restrictions. when enabled, files will also have readable filenames that otherwise would be gibberish.

anonymous traffic analytics ⊘

don't contribute to analytics

anonymous traffic analytics are needed to get an approximate number of active cobalt users. no identifiable information about you is ever stored. all processed data is anonymized and aggregated.

debug ⊘

enable features for nerds

gives you easy access to app info that can be useful for debugging. enabling this does not affect functionality of cobalt in any way.
